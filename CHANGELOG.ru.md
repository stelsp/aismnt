# Журнал изменений

## v2.0 — Автоматизация пайплайна

Этот релиз переводит `agent-kit` от ручного copy-paste workflow к автоматизированной оркестрации через Cursor субагентов.

### Ломающие изменения

- Удалены команды: `init`, `start`, `orchestrate`, `next`, `advance`, `complete`, `handoff`, `reopen`, `isolate`.
- Удален `team/handoff-template.md` и заменен на role-specific промпт-шаблоны.
- Стейджи переименованы: `planning` → `plan`, `implementation` → `implement`.
- Стейджи удалены: `fix` (имплементер теперь сам чинит при FAIL), `closure` (покрывается командой `validate`).

### Упрощение CLI (671 → 155 строк)

- Набор команд сокращен до `bootstrap`, `status` и `validate`.
- Оркестрация пайплайна перенесена из логики скрипта в AI-агента через `rules/orchestrator.mdc`.
- `bootstrap` теперь использует `team/task-template.md` по умолчанию, если `--task-file` не задан.
- Удалена macOS-специфичная зависимость `pbcopy`.

### Модель стейджей (4 вместо 6)

### Role-specific промпт-шаблоны

Добавлены отдельные шаблоны в `team/prompts/` для каждого стейджа:

- `plan.md` — таблица шагов плана, таблица тест-плана, риски и открытые вопросы.
- `implement.md` — таблица изменений, self-check чеклист и секция ответов на findings (fix-цикл).
- `review.md` — таблица findings с моделью severity и строгой изоляцией (нет доступа к плану).
- `qa.md` — таблица валидации acceptance criteria, таблица browser-шагов и edge cases.

Каждый шаблон использует `{{placeholders}}`, которые оркестратор подставляет перед запуском субагента.

### Правило оркестратора (`rules/orchestrator.mdc`)

Добавлено Cursor rule, которое автоматизирует весь пайплайн:

- Читает промпт-шаблоны и формирует stage-specific промпты.
- Запускает изолированные Task-субагенты для каждого стейджа.
- Парсит вердикты стейджей из handoff-файлов.
- Обрабатывает FAIL → re-implement → re-review/re-test циклы (максимум 2).
- Обновляет `status.json` после каждого перехода между стейджами.
- Запускает `validate` после завершения пайплайна.

### Влияние на workflow

До (v1): ~36 ручных действий на задачу (скопировать промпт, открыть чат, вставить, выполнить, сохранить handoff, запустить handoff-команду, повторить для 6 стейджей).

После (v2): 3 действия — запустить `bootstrap`, заполнить `task.md`, отправить одно сообщение в чат.

### npm scripts

```
agent:bootstrap      Создать новый pipeline run
agent:status         Показать состояние пайплайна
agent:validate       Проверить Definition of Done
agent:rules:install  Скопировать rules в .cursor/rules/
```
